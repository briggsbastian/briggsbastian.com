Kubernetes shows up on nearly every job posting in the space I'm aiming at,
and I can't honestly say I know it yet. The plan is to learn it the hard way,
following Kelsey Hightower's "Kubernetes the Hard Way": stand up the control
plane, etcd, the kubelets, and the PKI between them by hand, so the
abstractions have something underneath them when they leak.

I haven't started yet, so this note is mostly a placeholder. The real entries
come once I've actually built it: where the machinery surprises me, and how it
compares to the declarative model I already use with NixOS.
